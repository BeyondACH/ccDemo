## 1. 需求总结

用户需要一个**Python实现的6位数字加密/解密功能**，核心目标是对6位数字（如验证码、PIN码、ID片段等）进行安全加解密处理。

作为 系统开发者

以便于 安全地传输或存储敏感数字信息（如验证码、PIN码）
并确保 只有持有密钥的授权方能够还原原始数字

## 4. 验收标准（Given/When/Then）

**AC1：基本加密功能**
```
Given 一个有效的6位数字输入 "123456"
And 一个有效的加密密钥
When 调用加密函数
Then 返回一个非明文的加密字符串
```

**AC2：解密还原正确性**
```
Given 一个有效的加密字符串
And 对应的解密密钥
When 调用解密函数
Then 返回原始的6位数字
```

**AC3：输入校验**
```
Given 一个非6位数字的输入（如"12345"或"abcdef"）
When 调用加密或解密函数
Then 抛出明确的参数校验异常
```

**AC4：密钥错误处理**
```
Given 一个加密字符串
And 一个错误的解密密钥
When 调用解密函数
Then 解密失败并抛出密钥错误异常（不返回错误数据）
```

**AC5：边界值处理**
```
Given 边界值输入 "000000" 或 "999999"
When 调用加密后解密
Then 正确还原为原始输入值
```

## 7. 风险与假设

**假设：**
- 输入始终为纯数字字符串（非整数类型）
- 密钥由调用方安全存储和管理
- 不考虑量子计算攻击场景

**风险：**
| 风险 | 缓解措施 |
|------|----------|
| 密钥泄露导致数据暴露 | 建议密钥轮换机制（设计时预留接口） |
| 固定密钥加密可被重放攻击 | 可考虑加入时间戳或随机IV |
| 6位数字熵较低，易被暴力破解 | 建议加入尝试次数限制（由调用方实现） |

## 8. 交给设计代理的输入

**功能点清单：**
1. `encrypt(six_digit: str, key: str) -> str` - 加密函数
2. `decrypt(ciphertext: str, key: str) -> str` - 解密函数
3. 输入校验模块（6位纯数字校验）
4. 异常体系（参数错误、密钥错误、解密失败）

**候选算法方案（供设计代理评估）：**

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **AES-256-GCM** | 安全性高，业界标准 | 需处理IV，输出较长 | 高安全要求 |
| **Fernet (对称加密)** | Python标准库支持，使用简单 | 依赖cryptography库 | 通用场景 |
| **自定义置换+XOR** | 极简，输出可控长度 | 安全性较低 | 仅供混淆，非安全场景 |
| **Format-Preserving Encryption (FPE)** | 输出仍为6位数字 | 算法较复杂 | 需要保持格式的场景 |

**关键约束：**
- Python 3.7+ 兼容
- 依赖最小化原则
- 密钥不硬编码在代码中
